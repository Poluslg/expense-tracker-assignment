# Daybook — Frontend Assessment

Daybook is a tiny expense tracker built with Next.js (App Router) and TypeScript. You can add an expense, see your list, search it, and view a running total. It builds and runs.

It also has the kind of rough edges that real products ship when nobody is thinking about the person using them.

## Your task

Fork this repo, run it, and **actually use it.** Add an expense. Add a few. Try to add one without filling everything in. Search for something. Delete a row. Look closely at what shows up on screen.

Then ship a set of improvements as **separate, focused commits** (aim for around 10, one logical change each).

## What we care about

This is not a speed test and it is not about how much you can offload to an AI. We want to see your judgement.

- Did you notice the things that make the app annoying or confusing to use, not just the things that are visibly broken?
- When something is wrong, did you fix the actual cause or just hide the symptom?
- Is your code correct and clean? We will read it.
- Are your commits small, sensible, and clearly explained?

Roughly speaking we are looking at three things:

1. **Product and usability sense.** Does this feel like something built by someone who thought about the user?
2. **Code correctness.** Is the React and the data handling actually right?
3. **Communication.** Do your commits and notes explain your thinking?

## Rules

1. One logical change per commit. No single giant "improvements" commit.
2. **Write real commit messages.** A short summary plus a line or two on what was wrong and why your fix is right. This matters as much as the code.
3. Keep the stack. Do not swap frameworks or pull in a big UI library to do the work for you.
4. You may use AI tools. We assume you will. That is exactly why the easy stuff is not the point.
5. Keep a short **`FINDINGS.md`**: list what you found, what you changed, and anything you noticed but deliberately chose not to fix, with a reason. Knowing what to leave alone is part of the job.

## Running it

```bash
npm install
npm run dev
```
